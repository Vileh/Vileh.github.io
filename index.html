Pricing Algorithm in Python

OVERVIEW

This project was done as part of a University College Lodnon and University of California, San Diego research effort. I contributed as an independent Python programmer tasked with writing an an automated price optimization algorithm.

My code is available on

TASK

I was tasked to write a pricing algorithm for an (anonymous) product available in different local markets whose prices are set on a daily basis. The data given contained information for a set of representative markets: a history of prices, sales, and cost of sales for the firm whose prices needed to be optimized, as well as an average of competitor prices and some vectors of market conditions. The algorithm was expected to recommended next-period prices that maximize:

total firm profits = total sales x (price - per-unit cost).

RESULTS

When running the algorithm on the available data, we can calculate the expected profit when following the pricing algorithm. We were not given a validation set, so these results must be taken with a grain of salt. I managed to achieve an average increase of 40.49% over the historical profits. The estimated % increase in profits can be seen in the following graph:

As the competitor price is unknown at the time of picking our own price, the algorithm takes as inputs only the market conditions, our business's cost, and a bool variable signalling whether it is a weekday or the weekend. Here is a visualisation of the optimal price as a function of the market conditions (a higher number signifies worse market conditions):

The price returned by the algorithm is significantly lower for weekends as opposed to weekdays. This is due to the fact that demand is lower on weekends, and the algorithm is trying to maximize profits by capturing a larger market share. On the other hand, the price is higher during the weekdays as this way higher profits can be obtained from the price-inelastic customers.

CREDIT CARD APPROVAL: CLASSIFICATION WITH RANDOM FOREST

OVERVIEW

This project was done for the final assessment of MTH786P - Machine Learning with Python at Queen Mary University of London. The goal was to implement a binary classifier to predict the outcome of credit card applications using a dataset of approximately 1500 entries.

My code is available on

A detailed report on my work for this project can be found here.

TASK

The main challenge was to build a Random Forest classifier to predict credit card application outcomes while addressing significant class imbalance in the dataset. This involved preprocessing the data, implementing the Random Forest algorithm from scratch, and exploring techniques to improve performance on imbalanced data. The dataset required extensive preprocessing, including handling missing values and feature engineering.

IMPLEMENTED TECHNOLOGIES

The project involved implementing several key components:

  • Custom Random Forest Classifier: Implemented from scratch, including decision trees with various impurity measures (entropy, Gini index, misclassification error) and bootstrap sampling
  • Hyperparameter Tuning: Number of trees, max features, max depth, min samples per split, and decision threshold
  • K-Fold Cross-Validation: For robust model evaluation, using 4 splits to ensure adequate representation of the minority class
  • SMOTE (Synthetic Minority Over-Sampling Technique): To address class imbalance by creating synthetic samples of the minority class
  • F1 Score: As an evaluation metric for imbalanced classification, providing a better measure of model performance than accuracy alone

As an additional challenge, I implemented SMOTE based on the original paper by Chawla et al. This technique creates synthetic samples from the minority class to balance the dataset, which is particularly useful given the 9:1 ratio of accepted to rejected applications in our dataset.

RESULTS

I conducted several experiments to optimize the model's performance:

Tree Size Comparison:

This experiment compared four different tree sizes, showing that larger trees performed better, with the medium-sized model (max_features=7, max_depth=15, min_samples_split=2) achieving the best balance of performance and complexity.

Decision Threshold:

This experiment explored the effect of adjusting the decision threshold to address class imbalance. Lower thresholds biased the model towards predicting rejections, potentially improving recall at the cost of precision.

SMOTE and Decision Threshold:

The final experiment combined SMOTE with different decision thresholds. The best performing model achieved an accuracy of 0.93 and an F1 Score of 0.64, using SMOTE (N=2 or N=4) with the default decision threshold. Interestingly, combining SMOTE with a lower decision threshold (0.2) led to overprediction of rejections, resulting in lower overall performance.

KEY FINDINGS

  • Deeper trees performed better for this dataset, suggesting complex decision boundaries
  • SMOTE improved F1 Score without sacrificing accuracy, effectively addressing the class imbalance
  • The combination of SMOTE and adjusted decision thresholds requires careful tuning to avoid bias
  • The introduction of artificial samples significantly improved model performance, increasing the F1 Score from 0.43 to 0.64

SKILLS DEVELOPED

  • Machine Learning: Implementation of Random Forest algorithm from scratch, including various impurity measures
  • Data Preprocessing: Handling missing values, feature engineering, and addressing class imbalance

  • Model Evaluation: Using K-fold cross-validation and F1 Score for robust assessment of imbalanced data classification

  • Experimental Design: Planning and executing a series of experiments to systematically improve model performance

The Three-Body Problem: Stellar System Simulation

OVERVIEW

This project was completed as part of the MTH739P Computational Methods module at Queen Mary University of London. The task was to create a Python simulation of coplanar stellar systems with up to three stars, modeling various scenarios including stable orbits, chaotic behavior, and star mergers.

My code is available on

KEY FEATURES

  • Simulation of two-body and three-body systems
  • Implementation of both point-like stars and "hypergiants" with collision mechanics
  • Visualization of orbital trajectories and phase diagrams
  • Comparison of two integration techniques: scipy's solve_ivp() and the midpoint method
  • Convergence analysis of the numerical methods

IMPLEMENTATIONS

1. Two-Body System:

Implemented a stable two-body system with stars of mass ratio 1:2. The system demonstrates elliptical orbits and conservation of total momentum.

2. Three-Body Choreographies:

Simulated various three-body configurations, including:

  • Cris Moore's figure-eight solution
  • The "Butterfly III" configuration
  • The "Yin-Yang 2a" orbit
3. Hypergiant Simulations:

Modeled systems of hypergiants with collision mechanics. These simulations demonstrate star mergers and the conservation of mass and momentum during collisions.

NUMERICAL METHODS

The project involved implementing and comparing two numerical integration methods to solve the system of ordinary differential equations (ODEs) governing the stellar motions:

1. scipy's solve_ivp():

This built-in solver from SciPy uses adaptive step size control. For this project, the RK45 method (Dormand-Prince method) was employed, which is a fourth-order method with a fifth-order error estimator. Key features include:

  • Automatic step size adjustment for optimal balance between accuracy and computational efficiency
  • High accuracy, especially for smooth problems
  • Robust handling of stiff ODEs
2. Midpoint Method:

A custom implementation of the second-order Runge-Kutta method was developed. This method:

  • Uses a fixed step size
  • Provides a good balance between simplicity and accuracy
  • Is particularly well-suited for problems where the forces are velocity-dependent

The midpoint method was implemented as follows: 

y(t + h) = y(t) + h * f(t + h/2, y(t) + (h/2) * f(t, y(t)))

Where y is the state vector, t is time, h is the step size, and f is the ODE function.

Comparison and Convergence Analysis:

A detailed comparison between solve_ivp() and the midpoint method was conducted:

  • solve_ivp() showed superior accuracy, especially for larger step sizes
  • The midpoint method required significantly smaller step sizes to achieve comparable accuracy
  • solve_ivp() handled the rapid changes in acceleration near close encounters more effectively

A convergence analysis was performed to verify the second-order accuracy of the midpoint method:

  • Multiple simulations were run with decreasing step sizes: 0.1, 0.05, 0.025, and 0.0125
  • The error was calculated as the Euclidean norm of the difference between solutions at consecutive step sizes
  • The error was plotted on a log scale against step size
  • For the shorter time interval (max_time = 50), the error decreased by approximately a factor of 4 when halving the step size, confirming second-order convergence
  • For longer simulations (max_time = 250), the convergence rate was less consistent, likely due to accumulated errors over many orbits

This analysis highlighted the trade-offs between accuracy and computational cost, and demonstrated the importance of choosing appropriate numerical methods and step sizes for different types of simulations.

SKILLS DEVELOPED

  • Python programming: Object-oriented design, numerical methods implementation
  • Scientific computing: Solving ordinary differential equations, handling N-body problems
  • Data visualization: Creating informative plots of orbital trajectories and phase diagrams
  • Physics simulation: Modeling gravitational interactions and collision mechanics
  • Numerical analysis: Comparing integration methods and analyzing convergence

CONCLUSION

This project provided a deep dive into computational physics, combining theoretical knowledge of celestial mechanics with practical programming skills. The successful implementation of various stellar configurations and the analysis of numerical methods demonstrated a strong grasp of both the physical concepts and computational techniques involved in simulating complex dynamical systems.

My code is available on

KEY FEATURES

  • Movie review writing and storage
  • Watch list management
  • Data visualization of user's movie watching habits
  • Integration with IMDb Movies Dataset

IMPLEMENTATIONS

1. Writing a Review:

Users can select a movie from a drop-down menu, specify the review date, provide a score (1-10), and write their review. The app checks for existing reviews and updates them if necessary.

2. Managing Watch List:

Users can add movies to their watch list. The app prevents adding movies that have already been reviewed.

3. Data Visualization:

The app provides various visualizations to help users analyze their movie-watching habits:

  • Scatter plot comparing user scores to IMDb ratings
  • Bar chart of most-watched movie genres
  • Line chart showing the number of movies watched monthly

TECHNICAL IMPLEMENTATION

Database Structure:

The app uses SQLite to create and manage three tables:

  • movies: Stores information from the IMDb dataset
  • reviews: Stores user-written reviews
  • watchlist: Manages the user's watch list
Streamlit UI:

The user interface is built using Streamlit, providing an interactive and responsive experience. The app is divided into several tabs for different functionalities:

  • Write a Review
  • Add to Watch List
  • View Reviews
  • View Watch List
  • Delete Entries
  • Analysis

SKILLS DEVELOPED
